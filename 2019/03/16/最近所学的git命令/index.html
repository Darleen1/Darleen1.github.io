最近所学的git命令

初始化一个Git仓库，使用git init命令。
添加文件到Git仓库，分两步：
使用命令git add ，
使用命令git commit -m “”
git differ用来查看修改了什么
git status查看仓库状态
git log显示从最近到最远的提交日志
git reset可以把暂存区的修改退回到工作区
git reset –hard commit-id回退到以前的任意版本
git reflog查看命令历史
二次修改文件需要git add 两次 再git commit
Cat命令用于查看当前文件的内容
git checkout –撤销文件修改
git checkout 切换分支
rm删除工作区文件
git rm 删除版本库文件并且git commit -m “”
git remote add origin git@GitHub 关联仓库
git push -u origin master 把本地仓库所有内容推送到远程仓库，并关联分支
git push origin master 推送master分支最新的修改
git clone git账号 ：克隆本地仓库
git checkout -b issue-…
git stash apply恢复，stash内容并不删除，用git stash drop来删除；
git stash pop，恢复的同时把stash内容也删了。
rebase操作可以把本地未push的分叉提交历史整理成直线；
git branch -D强行删除分支
git pull把最新的提交从origin/dev抓下来，用于与小伙伴的最新提交和你试图推送的提交有冲突，然后，在本地合并，解决冲突，再推送
git tag 用于新建一个标签，默认为HEAD，也可以指定一个commit id；
git tag -a -m “blablabla…”可以指定标签信息；
git tag可以查看所有标签。
git push origin ..推送一个本地的标签
git push origin –tags推送全部未推送的本地标签
git tag -d ..删除本地标签
git push origin :refs/tags/..删除远程标签
