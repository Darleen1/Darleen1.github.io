我这两周

刚学git

  1. 一关机我就完全找不到头绪了，按照教程把以前的命令又打了一遍当然是不行的，错误百出，然后我知道了要从文件中打开git bash，然后可以继续学习了。
  2. 用记事本编辑文本，出现乱码。突然看错以为是要求用记事本编辑，原来是不要用记事本编辑，嗯，我下载了vscode。好了。
  3. git commit命令经常出错，有时还没在本地编辑就提交，有时在分支提交没合并，有时删除了又提交，提醒我找不到文件，粗心大意的。例如这种

    中间阶段

  • 先学搭建博客的，因为比较有趣。用GitHub搭建的

    这个在密钥上耽误了好长时间，看教程也看不大懂，又自己百度综合各种教程，其中包括这个。配置好了之后又要初始化，在git中比较慢
    so,我知道了cmd这个东西，好像真的比Git下载快哦。

  • 然后本人迫不及待的写了一篇博客perfect，无奈因为language没改，全是乱码。然后又百度了，改成了中文zh-Hans。
  • 学到时光穿梭，和远程仓库时，问题不大。因为搭建博客时一些准备工作都做好了。
  • 分支管理就比较难了
  1. 解决冲突时，我按照教程直接在本地改，可还是有冲突，后来我直接退出。我又把本地的文档改了，再用git bash打开又重新提交到远程仓库，就好了。
  2. no-ff模式，可以看出分支合并历史，一开始没用，后来用了，发现很乱。应该是因为有分隔符告诉我怎么在哪个分支修改。

    尾声

  • 实不相瞒，我也git clone了那个女装的，还浪费了我好长时间，发现不适合我，然后就从本地删除，从远程仓库删除了。
  • fork与clone分不清，fork只是在远程仓库的操作，clone克隆到本地

    balabala后面我的电脑住院了，去报修了。

  • markdown实在手机上看的，语法挺简单的，我还下载了简书，好像可以在简书上写markdown文章，但是我研究了还不会。
